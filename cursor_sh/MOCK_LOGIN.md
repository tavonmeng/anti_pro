# 模拟登录功能使用说明

## 测试账号

系统已启用模拟登录功能，可以使用以下测试账号进行登录：

### 普通用户账号
- **用户名**: `user`
- **密码**: `123456`
- **角色**: 选择"普通用户"

### 管理员账号
- **用户名**: `admin`
- **密码**: `123456`
- **角色**: 选择"管理员"

## 使用方式

1. 访问登录页面 (`/login`)
2. 选择对应的角色（普通用户或管理员）
3. 输入对应的用户名和密码
4. 点击"登录"按钮

## 模拟功能说明

### 登录功能
- 模拟登录会先尝试调用真实API，如果失败则使用模拟登录
- 模拟登录会验证用户名和密码
- 登录成功后会返回token和用户信息

### 任务管理功能
- 获取任务列表：返回预设的模拟任务数据
- 创建任务：创建新任务并添加到任务列表
- 更新任务：更新任务信息
- 删除任务：从任务列表中删除任务
- 更新状态：修改任务状态

### 模拟数据
系统预设了5个示例任务，包括不同状态的任务：
- 待处理 (pending)
- 进行中 (in_progress)
- 已完成 (completed)

## 切换真实API

如果需要切换到真实API，请修改 `src/utils/api.ts` 文件：

```typescript
// 将 ENABLE_MOCK 设置为 false
const ENABLE_MOCK = false
```

## 注意事项

1. 模拟功能仅在开发环境使用，生产环境应使用真实API
2. 模拟数据存储在内存中，刷新页面后会重置
3. 创建、更新、删除的任务会立即反映在界面上，但刷新后会恢复为初始数据
