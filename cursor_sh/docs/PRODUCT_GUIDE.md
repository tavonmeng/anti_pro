# AI设计任务管理系统 - 产品说明文档

## 📋 产品概述

### 产品定位

AI设计任务管理系统是一款面向 **VR+AI 裸眼3D内容定制** 行业的专业订单管理平台，为内容制作公司提供从订单接收、制作管理、客户反馈到项目交付的全流程数字化解决方案。

### 目标用户

1. **内容制作公司** - 主要服务对象
   - 接收和管理客户订单
   - 分配制作任务
   - 跟踪项目进度
   - 管理客户反馈

2. **企业客户** - 内容需求方
   - 在线下单
   - 查看制作进度
   - 预览作品
   - 提交修改意见

3. **制作团队** - 内容创作者
   - 接收制作任务
   - 上传作品预览
   - 响应客户反馈

### 核心价值

- ✅ **流程标准化** - 规范从下单到交付的全流程
- ✅ **状态可视化** - 实时掌握项目进展
- ✅ **沟通高效化** - 在线反馈，减少沟通成本
- ✅ **管理智能化** - 自动通知，防止遗漏

## 🎯 核心功能

### 一、用户角色体系

系统设计了三种角色，各司其职：

#### 1. 普通用户（客户）- `user`

**权限范围：**
- 注册和登录系统
- 创建三种类型的订单
- 查看自己的订单
- 预览制作成果
- 提交反馈意见（确认/修改）
- 接收邮件通知

**典型场景：**
> 某商场需要定制一套裸眼3D广告内容，市场部经理小王注册为用户，提交订单，在线查看制作进度，收到初稿后提出修改意见，最终确认验收。

#### 2. 负责人（制作人员）- `staff`

**权限范围：**
- 查看分配给自己的订单
- 上传预览文件（初稿/终稿）
- 更新订单状态
- 查看客户反馈

**典型场景：**
> 设计师张三被分配了一个数字艺术订单，他查看客户需求，制作完初稿后上传预览，收到客户修改意见后进行调整，最终提交终稿。

#### 3. 管理员 - `admin`

**权限范围：**
- 查看所有订单
- 分配订单负责人
- 管理负责人账户
- 更新订单状态
- 处理异常情况（取消订单等）

**典型场景：**
> 项目经理李经理每天登录系统，查看新订单，根据工作量和专长分配给合适的设计师，监控项目进度，处理客户投诉。

### 二、三种订单类型

系统支持三种不同的内容定制服务：

#### 1. 裸眼3D成片购买适配 (`video_purchase`)

**业务场景：**
客户已有现成的3D视频内容，需要适配到特定的裸眼3D显示设备。

**客户需要提供：**
- 行业类型（电影、户外广告、自定义）
- 视觉风格（科幻、写实、自定义）
- 视频时长（秒）
- 预算范围（最低价-最高价）
- 目标分辨率（如 3840x2160）
- 屏幕尺寸（如 55英寸）
- 屏幕曲率（如 1800R）

**交付内容：**
适配后的视频文件，可在指定设备上完美播放。

**典型场景：**
> 某影院购买了裸眼3D屏幕，需要将现有的科幻电影预告片适配为裸眼3D格式，在大厅播放以吸引观众。

---

#### 2. AI裸眼3D内容定制 (`ai_3d_custom`)

**业务场景：**
使用 AI 技术从零创作全新的裸眼3D内容。

**客户需要提供：**
- 显示设备配置信息（分辨率、尺寸等）
- 创意说明（主题、风格、要求）
- 现场实拍照片（展示环境和氛围）

**制作周期：** 5-7 个工作日

**交付内容：**
- 初稿预览（第3-4天）
- 终稿成片（第7天）

**典型场景：**
> 某科技展厅需要一个太空主题的裸眼3D内容，提供展厅照片和创意说明，AI生成符合要求的3D场景，经过客户确认后交付。

---

#### 3. 数字艺术内容定制 (`digital_art`)

**业务场景：**
定制数字艺术作品，用于商业空间、艺术展览等。

**客户需要提供：**
- 艺术方向（抽象、写实、装置艺术、动态艺术、自定义）
- 文字说明（描述期望的风格和感觉）
- 相关材料（参考图片、素材包等）

**制作周期：** 3 个工作日（初稿）

**交付内容：**
- 初稿（第3天）
- 修改稿（根据反馈）
- 最终成品

**典型场景：**
> 某咖啡馆想在墙面投影一幅抽象风格的数字艺术作品，营造艺术氛围。设计师根据咖啡馆提供的照片和风格要求，创作出符合空间特点的作品。

### 三、订单生命周期（状态机）

每个订单从创建到完成，都会经历一系列状态转换：

```
📝 创建订单
    ↓
1️⃣ 待分配 (pending_assign)
    ├─ 管理员查看订单详情
    ├─ 评估工作量和复杂度
    └─ 分配给合适的负责人
    ↓
2️⃣ 制作中 (in_production)
    ├─ 负责人查看需求
    ├─ 开始内容创作
    └─ 上传初稿预览
    ↓
3️⃣ 初稿预览 (preview_ready)
    ├─ 系统发送邮件通知客户
    ├─ 客户查看预览文件
    └─ 客户提交反馈
    ↓ 
    分支A: 客户确认通过
    ├─ 反馈类型: approval
    ├─ 自动转为"制作中"状态
    └─ 负责人继续制作终稿
    
    分支B: 客户要求修改
    ├─ 反馈类型: revision
    ├─ 自动转为"需要修改"状态
    ├─ 修改次数 +1
    └─ 负责人查看意见后重新制作
    ↓
4️⃣ 需要修改 (revision_needed)
    ├─ 负责人查看修改意见
    ├─ 重新制作
    └─ 上传新版本预览
    ↓ → 回到"制作中"状态
    
5️⃣ 制作中 (in_production) - 终稿阶段
    ├─ 负责人制作最终版本
    └─ 上传终稿预览
    ↓
6️⃣ 终稿预览 (final_preview)
    ├─ 系统发送邮件通知
    ├─ 客户查看终稿
    └─ 客户最终确认
    ↓
    确认通过 → 7️⃣ 已完成 (completed) ✅
    要求修改 → 回到"需要修改"状态
    
⚠️ 随时可能: 已取消 (cancelled)
    ├─ 仅管理员可操作
    └─ 订单终止
```

### 状态转换规则

| 当前状态 | 可转换的状态 | 触发条件 |
|---------|-------------|---------|
| 待分配 | 制作中、已取消 | 分配负责人、管理员取消 |
| 制作中 | 初稿预览、终稿预览、已取消 | 上传预览、管理员取消 |
| 初稿预览 | 制作中、需要修改、已取消 | 客户反馈、管理员取消 |
| 需要修改 | 制作中、已取消 | 负责人重新制作、管理员取消 |
| 终稿预览 | 已完成、需要修改、已取消 | 客户确认、要求修改、管理员取消 |
| 已完成 | 无 | 流程结束 |
| 已取消 | 无 | 流程结束 |

### 四、核心业务流程

#### 流程 1: 客户下单流程

```
1. 客户访问系统首页
   ↓
2. 点击"登录"按钮
   ├─ 已有账户 → 输入用户名/密码登录
   └─ 没有账户 → 点击"立即注册"
   ↓
3. 进入工作台，看到三个服务入口
   ├─ 裸眼3D成片购买适配
   ├─ AI裸眼3D内容定制
   └─ 数字艺术内容定制
   ↓
4. 选择需要的服务，点击"选购/定制"
   ↓
5. 填写订单表单
   ├─ 根据订单类型，填写不同的信息
   ├─ 上传必要的文件（照片、素材等）
   └─ 确认信息无误
   ↓
6. 点击"确认提交"
   ↓
7. 系统生成订单编号（如 ORD-20251105-A1B2）
   ↓
8. 显示成功提示
   ├─ AI定制: "预计5-7个工作日完成制作"
   ├─ 数字艺术: "预计3个工作日交付初稿"
   └─ 成片购买: "订单已提交，请等待制作"
   ↓
9. 返回"我的订单"页面
   ├─ 可以看到订单状态: "待分配"
   └─ 可以点击查看详情
```

**关键点：**
- 表单自动验证，防止漏填
- 文件上传有大小和格式限制
- 订单编号自动生成，格式统一
- 提交成功后立即可见

---

#### 流程 2: 管理员分配流程

```
1. 管理员登录系统
   ↓
2. 进入"订单管理"页面
   ├─ 可以看到所有订单
   ├─ 可以按状态、类型筛选
   └─ "待分配"的订单会突出显示
   ↓
3. 点击某个"待分配"订单
   ↓
4. 查看订单详情
   ├─ 订单类型和需求
   ├─ 客户上传的文件
   ├─ 预算和时间要求
   └─ 评估复杂度
   ↓
5. 点击"分配负责人"按钮
   ↓
6. 弹出负责人选择对话框
   ├─ 显示所有负责人列表
   ├─ 可以搜索负责人
   └─ 显示负责人当前工作量（可选）
   ↓
7. 选择合适的负责人，点击"确认"
   ↓
8. 系统自动：
   ├─ 订单状态变更为"制作中"
   ├─ 负责人收到新任务通知（邮件）
   └─ 订单详情显示负责人信息
```

**关键点：**
- 只有待分配状态才能分配
- 可以重新分配负责人
- 分配后自动进入制作状态
- 系统记录分配历史

---

#### 流程 3: 负责人制作流程

```
1. 负责人登录系统
   ↓
2. 进入"我的订单"页面
   ├─ 看到分配给自己的所有订单
   └─ 按状态和时间排序
   ↓
3. 点击"制作中"的订单
   ↓
4. 查看订单详情
   ├─ 客户需求说明
   ├─ 下载客户上传的文件
   ├─ 了解交付要求
   └─ 开始创作
   ↓
5. 制作完成后，准备上传预览
   ↓
6. 点击"上传预览文件"按钮
   ↓
7. 选择并上传文件
   ├─ 支持视频、图片等格式
   ├─ 可以上传多个文件
   └─ 显示上传进度
   ↓
8. 点击"提交预览"
   ↓
9. 更新订单状态
   ├─ 初稿 → 状态改为"初稿预览"
   └─ 终稿 → 状态改为"终稿预览"
   ↓
10. 系统自动：
    ├─ 发送邮件通知客户
    ├─ 邮件包含预览文件链接
    └─ 等待客户反馈
```

**关键点：**
- 只能操作分配给自己的订单
- 可以多次上传预览文件
- 上传后自动通知客户
- 系统记录所有版本

---

#### 流程 4: 客户反馈流程

```
1. 客户收到预览通知邮件
   ↓
2. 登录系统，进入"我的订单"
   ↓
3. 找到状态为"初稿预览"或"终稿预览"的订单
   ↓
4. 点击查看详情
   ↓
5. 查看预览文件
   ├─ 在线预览（视频/图片）
   ├─ 下载查看（大文件）
   └─ 评估是否满意
   ↓
6. 决定反馈类型
   
   路径A: 满意，确认通过
   ├─ 点击"确认通过"按钮
   ├─ 填写反馈内容（如："效果很好"）
   ├─ 点击"提交"
   └─ 系统自动：
       ├─ 初稿预览 → 制作中（制作终稿）
       └─ 终稿预览 → 已完成
   
   路径B: 需要修改
   ├─ 点击"需要修改"按钮
   ├─ 填写详细的修改意见
   │   例如："背景颜色改为蓝色调，
   │         飞船细节需要更精致，
   │         整体节奏可以更快"
   ├─ 点击"提交"
   └─ 系统自动：
       ├─ 状态变更为"需要修改"
       ├─ 修改次数 +1
       ├─ 通知负责人查看意见
       └─ 负责人收到邮件通知
   ↓
7. 反馈提交成功
   ├─ 显示提交时间
   ├─ 可以查看反馈历史
   └─ 等待负责人处理
```

**关键点：**
- 只能反馈自己的订单
- 必须填写反馈内容
- 可以多次提交修改意见
- 所有反馈都有记录

---

#### 流程 5: 修改和交付流程

```
【修改流程】
1. 负责人收到修改通知
   ↓
2. 登录系统，查看订单状态 = "需要修改"
   ↓
3. 点击订单，查看客户反馈
   ↓
4. 阅读修改意见，了解具体要求
   ↓
5. 更新订单状态为"制作中"
   ↓
6. 根据意见进行修改
   ↓
7. 修改完成后，上传新版本预览
   ↓
8. 再次等待客户反馈
   ├─ 如果还需修改，重复上述流程
   └─ 如果确认通过，进入终稿阶段

【交付流程】
1. 客户在"终稿预览"状态确认通过
   ↓
2. 系统自动将订单状态改为"已完成"
   ↓
3. 系统发送完成通知（邮件）
   ├─ 通知客户：订单已完成
   ├─ 通知负责人：任务完成
   └─ 通知管理员：项目结束
   ↓
4. 客户可以：
   ├─ 下载最终文件
   ├─ 查看完整的反馈历史
   └─ 评价服务（可选功能）
   ↓
5. 订单归档
   ├─ 状态不可再更改
   ├─ 保留所有历史记录
   └─ 用于后续分析和参考
```

### 五、通知机制

系统在关键节点自动发送邮件通知，确保信息及时传达：

#### 邮件通知场景

| 触发事件 | 通知对象 | 邮件内容 |
|---------|---------|---------|
| 订单创建成功 | 客户 | 订单编号、预计完成时间 |
| 订单被分配 | 负责人 | 新任务提醒、订单详情链接 |
| 状态变更 | 客户 | 从"待分配"→"制作中"等 |
| 预览文件上传 | 客户 | 初稿/终稿已就绪，请查看 |
| 客户提交反馈 | 负责人 | 客户已反馈，请查看意见 |
| 订单完成 | 客户、负责人、管理员 | 项目完成通知 |

#### 邮件模板示例

**初稿预览通知：**
```
主题：初稿预览已就绪 - ORD-20251105-A1B2

您好，

您的订单 ORD-20251105-A1B2 的初稿预览文件已上传。

请登录系统查看并提供反馈意见。

[查看预览] 按钮

此邮件由系统自动发送，请勿回复。
AI设计任务管理系统
```

### 六、权限控制

系统实现了细粒度的权限控制，确保数据安全：

#### 数据隔离规则

| 角色 | 可见订单范围 | 可操作范围 |
|------|------------|----------|
| 普通用户 | 仅自己创建的订单 | 创建订单、查看详情、提交反馈 |

- **管理员接口**: 负责人管理、订单分配
- **负责人接口**: 预览上传、状态更新

## 📊 业务指标

### 核心 KPI

1. **订单处理效率**
   - 订单平均完成时间
   - 首次响应时间（分配速度）
   - 初稿交付准时率

2. **客户满意度**
   - 一次性通过率（无需修改）
   - 平均修改次数
   - 订单完成率

3. **团队效能**
   - 负责人平均产能
   - 订单取消率
   - 客户投诉率

## 🚀 产品优势

### 相比传统方式的改进

| 传统方式 | 系统方式 | 改进效果 |
|---------|---------|---------|
| 邮件/微信沟通 | 在线订单系统 | 信息结构化，不遗漏 |
| Excel 管理订单 | 数据库存储 | 多人协作，实时同步 |
| 人工分配任务 | 系统辅助分配 | 工作量可视化 |
| 邮件发送预览 | 在线预览 | 版本管理，易查看 |
| 口头/文字反馈 | 结构化反馈 | 意见明确，可追溯 |
| 人工通知 | 自动邮件通知 | 及时性，无遗漏 |

### 可扩展性

系统设计了良好的扩展接口：

1. **新增订单类型** - 只需扩展表单和数据模型
2. **新增支付功能** - 预留支付接口
3. **对接第三方** - API 标准化，易集成
4. **数据分析** - 完整的数据记录
5. **移动端** - RESTful API，易开发 App
